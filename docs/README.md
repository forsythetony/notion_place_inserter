# Documentation

Material for this repository is grouped by intent. Each top-level folder includes an `archive/` subfolder for superseded pages you want to keep without mixing them into the active set.

| Section | Purpose |
|--------|---------|
| [technical-architecture](./technical-architecture/) | System design, phased delivery specs, runbooks, incidents, tech debt, SQL utilities |
| [feature-proposals](./feature-proposals/) | Product requirements and feature-level proposals |
| [marketing](./marketing/) | Positioning, messaging, and go-to-market notes |
| [market-research](./market-research/) | Competitive and vendor/library research |
| [style](./style/) | High-level product visual direction; canonical UI specs live in the frontend repo (linked from there) |
